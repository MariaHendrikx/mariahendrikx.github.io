## Why This Challenge?

Two years ago, the original plan was to build many startups, see whatever takes off and go for that one. However, I had done this already in 2019 on a small scale and after 2 weeks it felt a bit robotic. Also, the semester started a bit later again, so I wouldn't be able to sustain the focus.

So, this time, I just chose 1 topic of all the ideas I had and went all in on that one.

## The Challenge

So, what challenge am I starting? → **Build 100 startups. No time constraint.**

Obviously I don't want to spend 10 years on this challenge, so in the next weeks/months/years I will be posting about the progress of these startups.

The startups can be incredibly simple, but they can also be more complex.

## The Golden Rule

The golden rule behind every project is the following: **If you need a tutorial, I failed.**

Or in other words: the startups must be incredibly easy to use.

## The Rules

To keep myself accountable and ensure this challenge is meaningful, I've set some ground rules:

- **The project must be easy to use:** If you need a tutorial, I failed
- **It should be monetizable** - Not necessarily from day 1, but along the way
- **I'm allowed to work on builds together with other people**
- **Any tools allowed, any type of build allowed** - Can be a webpage, mobile app, or even hardware
- **It can be a teeny tiny mini-project or something super ambitious** - I can spend 20 minutes on it or multiple days/weeks
- **I must make a blog/report/reflection about each project** - And publish it online
- **I need to answer the same set of questions for each app** - Available in the report

## What to Expect

Throughout this challenge, I'll be sharing:

- Detailed blog posts about each app I build
- Code snippets and technical challenges I encounter
- Lessons learned and mistakes made
- Resources and tutorials that helped me along the way
- Progress updates and reflections on the journey
